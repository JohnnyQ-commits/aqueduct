# 需求：SaaS 用户周活跃留存分析

## 背景

SaaS 产品团队需要分析用户周活跃留存情况，用于评估产品粘性和用户健康度。

## 需求描述

基于用户会话数据，开发一张周粒度留存统计表，包含以下指标：

1. **统计周** (week_id) — 格式：YYYY-WXX
2. **周活跃用户数** (wau) — 当周有登录行为的去重用户数
3. **新增用户数** (new_users) — 当周首次登录的用户数
4. **周留存率** (retention_rate) — 连续两周活跃的用户占比
5. **平均会话时长** (avg_session_duration) — 当周所有会话的平均时长（秒）

## 数据来源

- 用户会话表：`dw_demo.dwd_user_session_di`
- 用户维度表：`dw_demo.dim_user_info_df`

## 统计口径

- **统计周期**：按周统计（ISO 周）
- **活跃定义**：当周至少有一次登录会话（`session_duration > 0`）
- **新增用户**：首次登录时间在当周的用户
- **留存用户**：当周和第前一周都有活跃行为的用户
- **分区过滤**：必须使用 `inc_day` 分区字段
- **数据范围**：最近 90 天

## 目标表设计

- **表名**：`dw_demo.dws_user_weekly_retention_di`
- **分区字段**：`inc_day` (STRING)
- **存储格式**：ORC
- **生命周期**：保留 365 天

## 特殊要求

1. **留存率计算**：使用自关联或窗口函数实现
2. **周格式**：使用 ISO 周格式（YYYY-WXX）
3. **边界处理**：首周无历史数据时留存率为 NULL

## 交付要求

1. DDL 建表语句
2. ETL SQL 开发（包含留存率计算逻辑说明）
3. 数据质量测试用例
4. 字段血缘图
5. 交付报告

## 验收标准

- SQL 通过代码审查
- 留存率计算逻辑正确（抽样验证）
- DQC 测试覆盖关键指标
