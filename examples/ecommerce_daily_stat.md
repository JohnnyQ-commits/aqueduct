# 需求：电商订单日活统计指标

## 背景

电商平台需要在数仓中新增一个订单日活统计报表，用于监控每日活跃订单数和成交金额趋势。

## 需求描述

基于现有订单数据，开发一张日粒度统计表，包含以下指标：

1. **统计日期** (inc_day)
2. **当日订单数** (order_count) — 去重统计
3. **当日成交客户数** (customer_count) — 去重统计
4. **当日GMV** (gmv) — 已支付订单的实付金额汇总
5. **当日客单价** (avg_order_amount) — GMV / 成交订单数

## 数据来源

- 订单主表：`dw_demo.dwd_order_info_di`
- 客户维度表：`dw_demo.dim_customer_info_df`

## 统计口径

- **统计周期**：按天统计
- **订单状态过滤**：仅统计已支付订单（`order_status >= '20'`）
- **分区过滤**：必须使用 `inc_day` 分区字段
- **去重逻辑**：订单数按 `order_id` 去重，客户数按 `customer_id` 去重

## 目标表设计

- **表名**：`dw_demo.dws_order_daily_stat_di`
- **分区字段**：`inc_day` (STRING)
- **存储格式**：ORC
- **生命周期**：永久保留
